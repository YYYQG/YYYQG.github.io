docker常用命令
2018/08/13 docker 常用命令 1988

关于docker

Docker 是一个开源的应用容器引擎，让开发者可以打包他们的应用以及依赖包到一个可移植的容器中，然后发布到任何流行的 Linux 机器上，也可以实现虚拟化。容器是完全使用沙箱机制，相互之间不会有任何接口。它是一种容器虚拟化技术。下面是我学习过程中使用的一些常用命令。

帮助命令

docker info
docker –help

镜像命令

docker images [options]
列出本地镜像
参数：
-a 列出所有的镜像（含中间镜像层 ）
-q 只显示镜像ID
-qa
–digests 显示镜像的摘要信息
–no-trunc 显示完整的镜像信息

docker search [options] 镜像名字
查找镜像（如Tomcat）
参数：
-s [数字] 如 -s 30 表示Stars超过30的
–no-trunc 显示完整的镜像描述
–automated 只列出automated build类型的镜像

docker history 镜像ID

docker pull 镜像名:Tag
下载镜像

docker rmi 某个镜像ID
删除镜像
参数：
-f 强制删除
docker rmi -f 镜像名1:Tag 镜像名2:Tag
删多个镜像
docker rmi -f $(docker images -qa)
删除所有镜像

容器命令

docker images [options]
列出本地镜像

-q 只显示镜像ID

docker history 镜像ID
查看指定镜像的创建历史
参数：
-H :以可读的格式打印镜像大小和日期，默认为true；
–no-trunc :显示完整的提交记录；
-q :仅列出提交记录ID。

下载镜像

docker rmi 某个镜像ID

-f 强制删除
docker rmi -f 镜像名1:Tag 镜像名2:Tag

//容器命令——————————————-
docker run [options] image [command][arg…]
新建并启动容器
options说明
–name “容器新名字”:为容器指定一个名称
-d:后台运行容器，并返回容器ID，也即启动守护式容器
-i:以交互模式运行容器，通常与-t同时使用
-t:为容器重新分配一个伪终端
-P:随机端口映射
-p:指定端口映射，有以下四种格式
ip:hostPort:containerPort
ip::containerPort
hostPort:containerPort
containerPort

docker ps [options]
列出当前运行的所有容器
参数：
-a:列出当前所有运行的容器+历史上运行过的
-l:显示最近创建的容器
-n:显示最近n个创建的容器
如 docker ps -n 2
-q:静默模式，只显示容器编号
–no-trunc:不截断输出

docker commit
提交容器副本使成为一个新的镜像
docker commit -m=”提交的描述信息” -a”作者” 容器ID 要创建的目标镜像名:[标签名]

exit
容器停止退出

ctrl+P+Q
容器不停止退出

docker start 容器名或ID
启动容器

docker restart 容器名或ID
重启容器

docker stop 容器名或ID
温柔停止容器

docker kill 容器名或ID
强制停止

docker rm 容器ID
删除未运行容器

docker rm -f $(docker ps -aq)
docker ps -aq | xargs docker rm
删除所有容器

重要

docker -d image
后台运行容器
docker ps 无法查看docker -d image 运行容器，因为它会自动退出，
docker ps -a能看见容器已经退出
docker容器后台运行，就必须有一个前台进程

docker logs -f -t –tail 容器ID
查看容器日志
参数：
-t 是加入时间戳
-f 跟随最新的日志打印
–tail [num] 如–tail 3 数字显示最后多少条日志

docker top 容器ID
查看容器内进程

docker inspect 容器ID
查看容器内部细节

docker exec -it 容器ID bashShell
在容器中运行某个命令，并显示结果
是在容器中打开新的终端，并且可以启动新的进程
docker exec -it 容器ID /bin/bash
进入正在运行的容器

docker attach 容器ID
进入正在运行的容器

docker cp 容器ID:容器内路径 目的主机路径
拷贝文件到主机
